# 世界构建器 (World Builder)

## 1. 模块概述

世界构建器负责从"世界种子"（小说、描述文档、配置等）自动推导并生成完整的世界模型。它是整个仿真系统的基石，决定了世界的物理规则、社会结构、实体网络等核心要素。

### 1.1 核心功能

```
世界种子
    │
    ▼
┌─────────────────────────────────────────────────┐
│                  世界构建器                      │
├─────────────────────────────────────────────────┤
│  1. 种子分析器 (Seed Analyzer)                   │
│     └─ 判断种子类型、提取核心要素                │
│                                                 │
│  2. 规则生成器 (Rule Generator)                 │
│     ├─ 物理规则库生成                            │
│     ├─ 非物理规则库生成                          │
│     └─ 社会规则库生成                            │
│                                                 │
│  3. 实体构建器 (Entity Builder)                 │
│     ├─ 实体类型识别                              │
│     ├─ 实体关系推断                              │
│     └─ 实体属性分配                              │
│                                                 │
│  4. 世界验证器 (World Validator)                │
│     └─ 检查世界一致性与合理性                    │
└─────────────────────────────────────────────────┘
    │
    ▼
世界模型（规则库 + 实体图 + 场变量）
```

### 1.2 输入输出

| 项目 | 类型 | 说明 |
|------|------|------|
| 输入 | WorldSeed | 世界种子，可以是文本、文件、结构化数据 |
| 输出 | WorldModel | 完整的世界模型，包含规则、实体、关系等 |

## 2. 种子分析器 (Seed Analyzer)

### 2.1 功能描述

分析输入的世界种子，判断世界类型，提取关键信息。

### 2.2 种子类型识别

```rust
pub enum SeedType {
    /// 现实世界背景（使用现实物理规则）
    RealWorld,
    /// 虚拟世界（需要自定义规则）
    VirtualWorld,
    /// 混合世界（部分现实规则 + 部分虚拟规则）
    HybridWorld,
    /// 未知类型（需要用户确认）
    Unknown,
}

pub struct SeedAnalysis {
    pub seed_type: SeedType,
    pub world_theme: String,          // 世界主题（如：武侠、科幻、末日）
    pub key_elements: Vec<String>,    // 关键元素列表
    pub entity_mentions: Vec<String>, // 提及的实体
    pub confidence: f32,               // 分析置信度
}
```

### 2.3 分析流程

```
输入种子
    │
    ▼
┌─────────────┐
│ 预处理       │  - 文本清洗
│             │  - 分词标注
└─────────────┘
    │
    ▼
┌─────────────┐
│ 规则匹配     │  - 使用规则库匹配世界类型
│             │  - 关键词提取
└─────────────┘
    │
    ▼
┌─────────────┐
│ LLM推理      │  - 对模糊内容进行语义分析
│             │  - 推断隐含信息
└─────────────┘
    │
    ▼
输出SeedAnalysis
```

### 2.4 实现方法

```python
class SeedAnalyzer:
    def __init__(self, llm_client, vector_db):
        self.llm = llm_client
        self.db = vector_db
        self.real_world_keywords = ["地球", "现代", "历史", "现实"]
        self.virtual_world_keywords = ["魔法", "修仙", "科幻", "末日", "虚拟"]

    def analyze(self, seed_text: str) -> SeedAnalysis:
        # 1. 关键词匹配
        keyword_score = self._match_keywords(seed_text)

        # 2. 向量检索相似世界
        similar_worlds = self.db.search(seed_text, top_k=5)

        # 3. LLM深度分析
        analysis_prompt = f"""
        分析以下世界种子，判断世界类型和提取关键信息：

        种子内容：{seed_text}

        请返回JSON格式的分析结果，包含：
        - world_type: real_world / virtual_world / hybrid / unknown
        - theme: 世界主题
        - key_elements: 关键元素列表
        - entities: 提及的实体类型
        - reasoning: 分析理由
        """
        llm_result = self.llm.generate(analysis_prompt)

        # 4. 综合判断
        return self._synthesize(keyword_score, similar_worlds, llm_result)
```

## 3. 规则生成器 (Rule Generator)

### 3.1 规则库结构

```rust
pub struct RuleLibrary {
    /// 物理规则库（重力、碰撞、时间等）
    pub physical_rules: PhysicalRules,
    /// 非物理规则库（魔法、超能力等）
    pub metaphysical_rules: Option<MetaphysicalRules>,
    /// 社会规则库（法律、习俗、道德等）
    pub social_rules: SocialRules,
    /// 经济规则库（货币、贸易、资源等）

}
```

### 3.2 物理规则生成

#### 3.2.1 现实世界规则

对于现实世界背景，使用预定义的物理规则库：

```yaml

  time_scale: 1.0           # 1现实秒 = 1游戏秒

    min: -273.15            # 绝对零度

  matter:
    collision_enabled: true
    friction_enabled: true
    decay_enabled: true

  biology:
    aging_enabled: true
    disease_enabled: true
    reproduction_enabled: true
```

#### 3.2.2 虚拟世界规则生成

对于虚拟世界，使用LLM根据种子内容生成规则：

```python
class RuleGenerator:
    def generate_virtual_rules(self, seed_analysis: SeedAnalysis) -> RuleLibrary:
        prompt = f"""
        基于以下世界信息，生成完整的世界规则库：

        世界类型：{seed_analysis.seed_type}
        世界主题：{seed_analysis.world_theme}
        关键元素：{seed_analysis.key_elements}

        请生成YAML格式的规则库，包含：
        1. 物理规则（基础运动、碰撞、时间流逝等）
        2. 超自然规则（如适用：魔法系统、科技系统等）
        3. 社会规则（政权结构、法律体系、社会阶层等）
        4. 经济规则（货币、资源分配、贸易机制等）

        规则需要具体、可计算、有明确的数值范围。
        """
        rules_yaml = self.llm.generate(prompt)
        return yaml.safe_load(rules_yaml)
```

### 3.3 规则验证

```python
class RuleValidator:
    def validate(self, rules: RuleLibrary) -> ValidationResult:
        issues = []

        # 1. 数值范围检查
        if rules.physical.gravity < 0:
            issues.append("重力不能为负数")

        # 2. 规则一致性检查
        if rules.physical.time_scale <= 0:
            issues.append("时间比例必须大于0")

        # 3. 规则完整性检查
        required_fields = ["gravity", "time_scale"]
        for field in required_fields:
            if not hasattr(rules.physical, field):
                issues.append(f"缺少必需字段: {field}")

        return ValidationResult(
            is_valid=len(issues) == 0,
            issues=issues
        )
```

## 4. 实体构建器 (Entity Builder)

### 4.1 实体类型定义

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EntityType {
    /// 实体类型名称
    pub name: String,
    /// 实体类型分类（人物、组织、地点、物品等）
    pub category: EntityCategory,
    /// 基础属性模板
    pub attributes: HashMap<String, AttributeSpec>,
    /// 行为能力
    pub capabilities: Vec<String>,
    /// 关系类型（可与其他实体建立的关系）
    pub relation_types: Vec<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum EntityCategory {
    /// 个人（如：村民、英雄、反派）
    Person,
    /// 组织（如：帮派、公司、军队）
    Organization,
    /// 地点（如：城市、建筑、区域）
    Location,
    /// 物品（如：武器、工具、装备）
    Item,
    /// 概念（如：事件、状态、法律）
    Concept,
}
```

### 4.2 实体识别与生成

```python
class EntityBuilder:
    def build_entities(self, seed_analysis: SeedAnalysis, rules: RuleLibrary) -> EntityGraph:
        entities = []

        # 1. 从种子中提取提及的实体
        mentioned_entities = self._extract_mentions(seed_analysis)

        # 2. 根据规则库推断需要的实体类型
        inferred_types = self._infer_entity_types(rules)

        # 3. 为每种类型创建实体模板
        for entity_type in inferred_types:
            template = self._create_entity_template(entity_type, rules)
            entities.append(template)

        # 4. 推断实体间的关系
        relations = self._infer_relations(entities, seed_analysis)

        # 5. 构建实体图
        return EntityGraph(
            entities=entities,
            relations=relations,
            rules=rules
        )

    def _infer_entity_types(self, rules: RuleLibrary) -> List[EntityType]:
        """根据规则库推断需要的实体类型"""
        prompt = f"""
        基于以下世界规则，推断这个世界需要哪些类型的实体：

        物理规则：{rules.physical_rules}
        社会规则：{rules.social_rules}
        经济规则：{rules.economic_rules}

        请返回实体类型列表，每种类型包含：
        - name: 类型名称
        - category: 分类（person/organization/location/item/concept）
        - attributes: 关键属性及其默认范围
        - capabilities: 该类型实体具备的行为能力
        """

        result = self.llm.generate(prompt)
        return json.loads(result)
```

### 4.3 实体属性分配

```python
class EntityFactory:
    def create_entity(self, template: EntityType, importance: float) -> Entity:
        """
        根据模板创建实体

        Args:
            template: 实体类型模板
            importance: 实体重要性（0.0-1.0），影响属性详细程度
        """
        entity = Entity(
            id=self._generate_id(),
            type=template.name,
            category=template.category,
        )

        # 根据重要性决定属性详细程度
        for attr_name, attr_spec in template.attributes.items():
            if importance > 0.8:
                # 重要实体：详细属性
                entity.attributes[attr_name] = self._generate_detailed_attribute(attr_spec)
            elif importance > 0.5:
                # 中等实体：基础属性
                entity.attributes[attr_name] = self._generate_basic_attribute(attr_spec)
            else:
                # 普通实体：简化属性
                entity.attributes[attr_name] = self._generate_simple_attribute(attr_spec)

        return entity
```

### 4.4 实体关系构建

```python
class RelationBuilder:
    def build_relations(self, entities: List[Entity], seed_analysis: SeedAnalysis) -> List[Relation]:
        relations = []

        # 1. 从种子中提取显式关系
        explicit_relations = self._extract_explicit_relations(seed_analysis)
        relations.extend(explicit_relations)

        # 2. 根据类型推断隐式关系
        implicit_relations = self._infer_implicit_relations(entities)
        relations.extend(implicit_relations)

        # 3. 随机化生成额外关系（填充空白）
        random_relations = self._generate_random_relations(entities, len(relations))
        relations.extend(random_relations)

        return relations

    def _infer_implicit_relations(self, entities: List[Entity]) -> List[Relation]:
        """根据实体类型推断隐式关系"""
        relations = []

        # 按类型分组
        by_category = defaultdict(list)
        for entity in entities:
            by_category[entity.category].append(entity)

        # 组织与个人的关系
        for org in by_category[EntityCategory.Organization]:
            for person in by_category[EntityCategory.Person]:
                # 随机分配隶属关系
                if random.random() < 0.3:  # 30%概率有关联
                    relations.append(Relation(
                        from_id=org.id,
                        to_id=person.id,
                        type=random.choice(["employs", "owns", "controls"]),
                        strength=random.random()
                    ))

        return relations
```

## 5. 世界模型 (World Model)

### 5.1 数据结构

```rust
pub struct WorldModel {
    /// 世界唯一标识
    pub id: String,
    /// 世界名称
    pub name: String,
    /// 世界类型
    pub world_type: WorldType,
    /// 规则库
    pub rules: RuleLibrary,
    /// 实体图
    pub entity_graph: EntityGraph,
    /// 场变量（宏观影响力）
    pub fields: Vec<FieldVariable>,
    /// 初始状态

    /// 元数据
    pub metadata: WorldMetadata,
}

pub struct EntityGraph {
    /// 实体节点
    pub nodes: HashMap<EntityId, Entity>,
    /// 实体关系
    pub edges: Vec<Relation>,
    /// 空间索引（用于快速查询）
    pub spatial_index: Option<SpatialIndex>,
}

pub struct FieldVariable {
    /// 场变量名称
    pub name: String,
    /// 变量类型
    pub var_type: FieldVarType,
    /// 初始值
    pub initial_value: f64,
    /// 传播速度
    pub propagation_speed: f64,
    /// 衰减系数
    pub decay_rate: f64,
    /// 影响范围
    pub influence_radius: f64,
}
```

### 5.2 世界验证

```python
class WorldValidator:
    def validate(self, world: WorldModel) -> ValidationResult:
        issues = []

        # 1. 规则验证
        rule_validation = RuleValidator().validate(world.rules)
        issues.extend(rule_validation.issues)

        # 2. 实体图连通性检查
        if not self._check_connectivity(world.entity_graph):
            issues.append("实体图不连通，存在孤立实体")

        # 3. 实体引用完整性
        for relation in world.entity_graph.edges:
            if relation.from_id not in world.entity_graph.nodes:
                issues.append(f"关系引用了不存在的实体: {relation.from_id}")

        # 4. 场变量合理性
        for field in world.fields:
            if field.decay_rate < 0 or field.decay_rate > 1:
                issues.append(f"场变量 {field.name} 的衰减系数不在[0,1]范围内")

        return ValidationResult(
            is_valid=len(issues) == 0,
            issues=issues
        )
```

## 6. 完整构建流程

```python
class WorldBuilder:
    def __init__(self, llm_client, vector_db):
        self.seed_analyzer = SeedAnalyzer(llm_client, vector_db)
        self.rule_generator = RuleGenerator(llm_client)
        self.entity_builder = EntityBuilder(llm_client)
        self.world_validator = WorldValidator()

    def build(self, seed: WorldSeed) -> WorldModel:
        # 1. 分析种子
        print(f"分析世界种子...")
        seed_analysis = self.seed_analyzer.analyze(seed.content)

        # 2. 生成规则库
        print(f"生成世界规则...")
        if seed_analysis.seed_type == SeedType.RealWorld:
            rules = self.rule_generator.load_real_world_rules()
        else:
            rules = self.rule_generator.generate_virtual_rules(seed_analysis)

        # 3. 构建实体
        print(f"构建实体网络...")
        entity_graph = self.entity_builder.build_entities(seed_analysis, rules)

        # 4. 初始化场变量
        print(f"初始化场变量...")
        fields = self._initialize_fields(rules, entity_graph)

        # 5. 组装世界模型
        world = WorldModel(
            id=self._generate_id(),
            name=seed.name,
            world_type=seed_analysis.seed_type,
            rules=rules,
            entity_graph=entity_graph,
            fields=fields,
            initial_state=self._create_initial_state(entity_graph, rules),
            metadata=WorldMetadata(
                created_at=datetime.now(),
                seed_source=seed.source,
                version="1.0.0"
            )
        )

        # 6. 验证世界
        validation = self.world_validator.validate(world)
        if not validation.is_valid:
            raise WorldBuildError(f"世界验证失败: {validation.issues}")

        print(f"世界构建完成！")
        print(f"  - 实体数量: {len(entity_graph.nodes)}")
        print(f"  - 关系数量: {len(entity_graph.edges)}")
        print(f"  - 场变量: {len(fields)}")

        return world
```

## 8. 扩展与优化

### 8.1 增量更新

世界构建器支持增量更新，当种子内容变化时只更新受影响的部分：

```python
class IncrementalWorldBuilder:
    def update(self, world: WorldModel, new_seed: WorldSeed) -> WorldModel:
        # 1. 分析新种子
        new_analysis = self.seed_analyzer.analyze(new_seed.content)

        # 2. 对比差异
        diff = self._compare_analysis(world.metadata, new_analysis)

        # 3. 增量更新
        if diff.rules_changed:
            world.rules = self.rule_generator.update_rules(world.rules, new_analysis)

        if diff.entities_changed:
            world.entity_graph = self.entity_builder.update_entities(
                world.entity_graph,
                diff.entity_changes
            )

        return world
```

### 8.2 多种子融合

支持从多个种子源融合生成世界：

```python
class MultiSeedFusion:
    def fuse(self, seeds: List[WorldSeed]) -> WorldModel:
        # 1. 分析所有种子
        analyses = [self.seed_analyzer.analyze(s.content) for s in seeds]

        # 2. 检测冲突
        conflicts = self._detect_conflicts(analyses)

        # 3. 解决冲突（通过LLM协商）
        resolved = self._resolve_conflicts(conflicts)

        # 4. 融合生成世界
        return self._build_from_resolved(resolved)
```
