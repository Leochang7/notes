# 世界引擎架构 - 总览

## 1. 系统概述

世界引擎是一个支持百万级智能Agent并行运行的分布式仿真系统，能够在虚拟世界中实现从微观个体到宏观社会的全层次仿真。系统采用分层架构，通过视界划分、状态降级、动态具象化等技术，在保证仿真实时性和一致性的同时，极大降低计算成本。

### 1.1 核心设计理念

| 原则 | 说明 |
|------|------|
| **按需精度** | 系统根据观察者的注意力分配计算资源，实现算力的阶梯式降级 |
| **无缝坍缩** | 不同精度的区域之间可以无缝转换，观察者无法感知精度差异 |
| **事件驱动** | 仿真由事件触发而非时钟驱动，减少无效计算 |
| **语义缓存** | 对相似查询进行语义级缓存，避免重复推理 |
| **蜂群思维** | 相近的Agent共享部分认知上下文，降低个体推理成本 |

### 1.2 系统能力指标

| 指标 | 目标值 |
|------|--------|
| 最大Agent数量 | 100万+ |
| 实时互动Agent | 100-1000个（聚光灯区） |
| 区域转换延迟 | <100ms |
| Agent响应延迟 | <50ms（本地模型） |
| 语义缓存命中率 | >80% |
| 内存占用（每Agent） | <10KB（非活跃状态） |

## 3. 核心概念

### 3.1 视界划分 (Horizon Partitioning)

根据系统的注意力分配，将虚拟世界划分为三个精度层：

| 区域 | 范围 | Agent数量 | 精度 | 计算方式 |
|------|------|-----------|------|----------|
| 聚光灯区 (Spotlight) | 玩家视野/观察焦点 | 100-1000 | 全精度 | LLM + 行为树 |
| 边缘区 (Penumbra) | 视距外但可能接触的区域 | 1000-10000 | 中精度 | 行为树 + 马尔可夫链 |
| 黑盒区 (Background) | 远程区域 | >100000 | 低精度 | 统计方程 + 场变量 |

### 3.2 状态具象化 (State Hydration)

当注意力从低精度区域转移到高精度区域时，将抽象的状态数据转换为具象的Agent状态。

```
宏观统计变量 ──▶ 实例化Agent ──▶ 注入记忆 ──▶ LLM推理 ──▶ 生成行为
```

### 3.3 场变量 (Field Variables)

将宏观影响力抽象为连续的空间场变量，像波纹一样在不同区域间传播。

```
黑盒区经济崩溃 ──▶ Economic_Panic_Index ↑ ──▶ 传播到聚光灯区 ──▶ 影响Agent决策
```

### 3.4 认知时钟 (Cognitive Tick)

不同区域的Agent有不同的认知时钟频率：

- 聚光灯区：1-10秒（复杂事件触发）
- 边缘区：10-60秒（定时触发）
- 黑盒区：分钟/小时级（宏观统计周期）

### 3.5 世界时钟 (World Clock) ⏰

将真实时间乘以加速系数k转换为模拟时间，支持快速运行大规模仿真。

| 加速系数 | 用途 | 示例 |
|---------|------|------|
| k=1 | 实时仿真 | 玩家直接互动 |
| k=60 | 中速加速 | 1分钟真实=1小时模拟 |
| k=1440 | 快速演化 | 1分钟真实=1天模拟 |
| k=10080 | 超快演化 | 1分钟真实=1周模拟 |
| k=525600 | 历史模拟 | 1天真实=1年模拟 |

## 4. 技术选型

### 4.1 核心技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 编程语言 | Rust / Go | 高性能、并发安全 |
| LLM推理 | vLLM / TGI | 高吞吐量推理服务 |
| 向量数据库 | Milvus / Qdrant | 语义检索与缓存 |
| 消息队列 | Kafka / RabbitMQ | 事件总线实现 |
| 知识图谱 | Neo4j / NebulaGraph | 世界模型存储 |
| 时序数据库 | InfluxDB / TDengine | 统计数据存储 |
| 缓存 | Redis | 热数据缓存 |

### 4.2 Agent模型选择

| 模型规模 | 参数量 | 用途 | 部署位置 | 推理延迟 |
|---------|--------|------|----------|----------|
| 本地小模型 | 0.5B - 3B | 本能反应、短对话 | 玩家设备/边缘节点 | <50ms |
| 云端中模型 | 7B - 14B | 深度决策、世界观演进 | 云端服务器 | 100-500ms |
| 云端大模型 | 70B+ | 复杂推理、世界构建 | 专用推理集群 | 1-5s |

## 5. 系统流程

### 5.1 初始化流程

```
1. 接收世界种子（小说/描述）
2. 分析种子类型（现实世界/虚拟世界）
3. 生成物理规则库 + 非物理规则库
4. 构建实体网络图（节点：实体，边：关系）
5. 实例化世界模型
6. 初始化区域管理器
7. 启动事件总线
8. 开始仿真循环
```

### 5.2 仿真循环

```
每一帧:
1. 收集事件（玩家输入、定时触发、区域边界事件）
2. 更新场变量（从黑盒区向聚光灯区传播）
3. 各区域按认知时钟更新Agent状态
4. 处理Agent请求（感知、决策、行动）
5. 执行物理交互
6. 检测区域转换，触发状态具象化/抽象化
7. 发送区域间事件
```

### 5.3 Agent认知循环

```
感知 (Perceive):
├─ 收集环境信息（场变量、附近实体、事件）
├─ 从记忆库检索相关历史
└─ 构建当前上下文

决策 (Decide):
├─ 本地小模型：快速反应
├─ 云端模型：深度思考
└─ 行为树：执行常规动作

行动 (Act):
├─ 生成行为意图
├─ 交由游戏引擎执行
└─ 更新自身状态和记忆
```

## 6. 非功能性需求

### 6.1 性能要求

- 聚光灯区Agent响应延迟 < 50ms
- 区域转换延迟 < 100ms
- 支持100万+Agent同时存在
- 语义缓存命中率 > 80%
- 单次LLM推理 < 2秒

### 6.2 可扩展性

- 支持水平扩展（增加服务器节点）
- 支持区域动态划分与合并
- 支持模型热更新

### 6.3 可靠性

- 故障转移时间 < 5秒
- 数据持久化保证
- 事件不丢失（至少一次投递）

### 6.4 可观测性

- 全链路追踪
- 实时性能监控
- Agent行为回放

## 7. 关键挑战与解决方案

| 挑战 | 解决方案 |
|------|----------|
| 计算资源限制 | 视界划分 + 状态降级 |
| Agent行为一致性 | 共享上下文 + 蜂群思维 |
| 区域转换突兀 | 状态具象化 + 场变量平滑过渡 |
| 大规模并发 | 异步事件总线 + 状态池化 |
| 语义理解成本 | 向量缓存 + 相似度聚合 |
| 记忆管理 | 长短期记忆分离 + 定期压缩 |
