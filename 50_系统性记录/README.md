## 1.模块作用
这个模块**不记录知识，也不记录项目**，  
它只做一件事：
> **维护我的长期状态与节奏。**

包括：
- 我在做什么
- 我做得怎么样
- 我要不要调整方向
## 2. 什么内容应该放在这里？
### ✅ 应该放的

- 计划（周 / 月 / 阶段）
- 复盘（事情做得如何）
- 长期清单（TODO / Checklists）
### ❌ 不应该放的

- 知识点
- 技术总结
- 项目设计细节
- 一次性灵感（先放 Inbox）

判断标准只有一句话：

> **这条记录，是在管理“我”，而不是管理“事”。**

## 3.原则
### 规则 1：频率比完整度重要
- 一句话也算记录
- 不强求格式
- 不因为“写不好”而不写
### 规则 2：允许失败记录

这里允许：
- 计划没完成
- 状态很差
- 决策是错的

因为这里的目标不是“展示”，  
而是**可追溯**。

这里只记录：

> **这段时间，我这个人运行得怎么样。**
