## 并发编程
### 什么是 Java 中的 CAS 机制？
**CAS（Compare-And-Swap）** 是一种乐观锁机制。它包含三个操作数：内存位置、预期原值和新值。执行时，只有当内存位置的值与预期原值相同时，才将该位置的值更新为新值，否则不执行更新。底层通过 CPU 的 **cmpxchg** 指令实现原子性。

## 集合
### Java 中 HashMap 的底层数据结构是什么？
**HashMap** 的底层数据结构在 JDK 1.8 及之后是**数组 + 链表 + 红黑树**。数组是 Node 类型的 table 数组，每个数组元素称为一个桶。当发生哈希冲突时，冲突的节点会以链表形式存储在同一个桶中。当链表长度超过阈值（默认为 8）且数组长度大于等于 64 时，链表会转换为红黑树，以提高查询效率。当红黑树节点数小于 6 时，会退化为链表。
